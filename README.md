# Gemini RAG System Walkthrough

## Overview
This is a comprehensive Retrieval-Augmented Generation (RAG) system built with **Gemini 2.5 Flash** and **Qdrant**. It features a modular multi-page Streamlit interface for seamless configuration, document ingestion, and intelligent chatting.

## Features
- **Modular Frontend**: Separation of concerns with Configuration, Knowledge Base, and Chat pages.
- **Multimodal capabilities**: Extracts and indexes images from PDFs (placeholder for full description generation enabled).
- **Hybrid Search**: Uses Qdrant for robust vector retrieval.
- **Citations**: Provides source page numbers and scores for every answer.

## How to Run
1. **Install Dependencies**:
   ```bash
   pip install -r requirements.txt
   ```
2. **Start the Application**:
   ```bash
   streamlit run main.py
   ```

## Usage Guide
1. **Configuration Page**:
   - Enter your **Gemini API Key** and **Qdrant URL/API Key**.
   - Save the configuration.
2. **Knowledge Base Page**:
   - Upload a PDF document.
   - Adjust chunk size.
   - Click **Process and Index**.
   - Wait for the success message (this indexes text and images).
3. **AI Chatbot Page**:
   - Ask questions about your document.
   - View answers generated by Gemini 2.5 Flash.
   - Expand the **Sources** section to verify citations.

## Technical Details
- **Text Chunking**: Recursive character splitting with overlap.
- **Image Handling**: `pymupdf` extracts images; system is ready to generate descriptions using Gemini Vision.
- **State Management**: Uses `st.session_state` to persist keys and chat history across pages.
